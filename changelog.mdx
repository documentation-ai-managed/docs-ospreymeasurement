## Recent Updates

Track the latest changes to Osprey Measurement Systems products and documentation. You receive notifications for major releases via your dashboard at `https://dashboard.example.com`. Review updates relevant to your deployed sensors like the Osprey Tilt String (`OTS`) and In-Place Extensometer (`IPX`).

<Callout kind="tip">
Subscribe to changelog updates in your account settings to get email notifications for new releases and critical fixes.
</Callout>

<Update label="2024-10-15" description="v2.1.0" tags={["feature", "improvement"]}>
## New Features

- Added support for up to 200 micro-tilts on the `OTS` sensor bus for higher resolution monitoring.
- Introduced wireless configuration endpoint for `IPX` devices via `https://api.example.com/v2/config`.

## Improvements

- Enhanced data export formats including CSV and JSON with timestamps.
- Optimized firmware for 20% lower power consumption during extended deployments.

## Bug Fixes

- Resolved intermittent connection drops on `OTS` in high-vibration environments like railways.
- Fixed calibration drift issue in `IPX` readings after 30+ days of operation.
</Update>

<Update label="2024-09-10" description="v2.0.0" tags={["feature", "breaking"]}>
## New Features

- Launched `OTS` Pro variant with embedded concrete mounting and IP68 rating.
- New API endpoint `/sensors/{id}/alerts` for real-time threshold notifications.

## Breaking Changes

- Updated authentication to require `Bearer {YOUR_TOKEN}` format. Legacy API keys deprecated.
- Sensor data schema now includes `precision` field; update your integrations.

## Bug Fixes

- Corrected tilt angle miscalculations on `OTS` during rapid temperature changes.
- Patched security vulnerability in firmware update process.
</Update>

<Update label="2024-07-25" description="v1.5.0" tags={["bugfix", "improvement"]}>
## Improvements

- Dashboard now supports custom sensor grouping for multi-site projects.
- Increased `IPX` sampling rate to 10Hz for dynamic load monitoring.

## Bug Fixes

- Fixed dashboard rendering issue with large `OTS` datasets (>150 points).
- Resolved webhook delivery failures to `https://your-webhook-url.com/webhook`.
</Update>

## Checking for Updates

Follow these steps to verify and apply the latest firmware to your devices.

<Steps>
  <Step title="Check Version" icon="check-circle">
    Log in to your dashboard and navigate to Sensors > Firmware.
    
````bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://api.example.com/v2/sensors/{id}/firmware
````
  </Step>
  <Step title="Download Update" icon="download">
    Select the latest version matching your device model (`OTS` or `IPX`).
  </Step>
  <Step title="Apply Firmware" icon="upload">
    Upload via the dashboard or API.
    
    <Callout kind="alert">
    Backup your calibration data before updating.
    </Callout>
  </Step>
</Steps>
