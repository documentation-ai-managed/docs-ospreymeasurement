# Outputs

The ASP communicates digitally via an RS485 BUS. Your data logger will recall measurements from the ASP via an ASCII protocol or MODBUS RTU.

The following provides details on each of the possible outputs:

## Voltage

**Voltage received by each sensor during measurement.&#x20;**

This provides useful troubleshooting metric, and often a better indicator of reducing battery power than the power recorded by the data logger itself.&#x20;

## Temperature

**Temperature at each sensor in degrees Celsius.&#x20;**

While temperature compensation is carried out during the *Distance&#x20;*&#x6D;easurement, the temperature reading confirms stable temperature across the system and identifies any physical issues that may be impacting the measurement performance.

## Status

**Numerical code indicating measurement status.**

| Status | Osprey Error Code | Description                      |
| ------ | ----------------- | -------------------------------- |
| 0      | N/A               | Measurement OK                   |
| 1      | -99               | Measurement Not Ready            |
| 2      | -98               | Pressure value outside threshold |
| 3      | -97               | Sensor reports an internal error |
| 4      | -96               | Insufficient power               |
