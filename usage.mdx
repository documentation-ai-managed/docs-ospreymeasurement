## Overview

Use these tools to access, interpret, and manage data from your Osprey Tilt String (OTS) and In-Place Extensometer (IPX) instruments. Automate data collection to reduce manual effort and enable real-time monitoring of geotechnical assets like retaining walls, pipelines, and diaphragm walls.

<Callout kind="tip">
Prepare your API key from the dashboard before starting. Visit `https://dashboard.example.com` to generate one.
</Callout>

## Accessing Data via Support Tools

Connect to your instruments using the web dashboard, CLI tool, or direct API calls. Choose the method that fits your workflow.

<Tabs>
  <Tab title="Web Dashboard" icon="monitor">
    Log in at `https://dashboard.example.com` and navigate to your project.

    <Steps>
      <Step title="Select Project">
        Choose your site from the dashboard list.
      </Step>
      <Step title="View Live Data">
        Click the `OTS` or `IPX` tab to see real-time readings.
      </Step>
      <Step title="Export Data">
        Download CSV files for analysis.
      </Step>
    </Steps>
  </Tab>
  <Tab title="CLI Tool" icon="terminal">
    Install the Osprey CLI for command-line access.

    <CodeGroup tabs="macOS,Linux,Windows">
      ```bash
      brew install osprey-cli
      ```
      ```bash
      curl -sSL https://get.osprey.tools | sh
      ```
      ```powershell
      winget install Osprey.CLI
      ```
    </CodeGroup>

    Authenticate and fetch data:

    ```bash
    osprey login --api-key YOUR_API_KEY
    osprey data fetch --project-id proj_123 --sensor OTS
    ```
  </Tab>
  <Tab title="API" icon="code">
    Use REST API for programmatic access.

    <Request tabs="cURL,Python" show-lines="true">
      ```bash
      curl -X GET "https://api.example.com/v1/projects/proj_123/sensors/ots/data" \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
      ```python
      import requests

      headers = {"Authorization": "Bearer YOUR_API_KEY"}
      response = requests.get(
          "https://api.example.com/v1/projects/proj_123/sensors/ots/data",
          headers=headers
      )
      print(response.json())
      ```
    </Request>
  </Tab>
</Tabs>

## Interpreting Tilt and Extensometer Readings

Tilt readings from OTS show angular displacement in microradians (`µrad`). Extensometer (IPX) measures strain in millimeters (mm). Compare against thresholds for alerts.

| Sensor Type | Parameter | Unit | Normal Range | Alert Threshold |
|-------------|-----------|------|--------------|-----------------|
| OTS        | Tilt     | `µrad` | `<50`       | `>200`         |
| IPX        | Extension| mm   | `<1`        | `>5`           |

<Columns cols={2}>
  <Card title="OTS Tilt Example" icon="trending-up">
    A reading of `125 µrad` indicates minor settlement. Plot over time for trends.
  </Card>
  <Card title="IPX Strain Example" icon="ruler">
    `2.3 mm` extension suggests soil movement. Cross-reference with adjacent sensors.
  </Card>
</Columns>

```javascript
// Sample OTS data parsing
const data = [
  { position: 1, tilt: 45 },
  { position: 2, tilt: 120 }
];
const maxTilt = Math.max(...data.map(d => d.tilt));
console.log(`Max tilt: ${maxTilt} µrad`);
```

## Scheduling Automated Measurements

Set up recurring data pulls to automate monitoring.

<Steps>
  <Step title="Configure Cron Job" icon="clock">
    Edit your crontab for hourly fetches.

    ```bash
    crontab -e
    # Add: 0 * * * * /path/to/fetch-data.sh proj_123
    ```
  </Step>
  <Step title="Use Webhook Alerts" icon="bell">
    In the dashboard, set thresholds and webhook URL: `https://your-webhook-url.com/webhook`.
  </Step>
  <Step title="Python Scheduler" icon="code">
    ```python
    import schedule
    import time

    def fetch_data():
        # API call here
        print("Data fetched")

    schedule.every().hour.do(fetch_data)
    while True:
        schedule.run_pending()
        time.sleep(60)
    ```
  </Step>
</Steps>

## Routine Maintenance and Troubleshooting

Perform checks monthly to ensure reliability.

<ExpandableGroup>
  <Expandable title="Basic Maintenance" default-open="true">
    - Inspect cables for damage.
    - Clean sensor housings.
    - Verify battery levels via dashboard.
  </Expandable>
  <Expandable title="Common Issues">
    No data? Check API key and network connectivity.

    High noise? Calibrate sensors:

    ```bash
    osprey calibrate --sensor-id ots_456 --project proj_123
    ```
  </Expandable>
  <Expandable title="Advanced Diagnostics">
    Run full system test:

    <Request show-lines="false">
      ```bash
      curl -X POST "https://api.example.com/v1/projects/proj_123/diagnostics" \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -d '{"sensors": ["ots_456", "ipx_789"]}'
      ```
    </Request>
  </Expandable>
</ExpandableGroup>
